WORKFORCE DAILY TRACKER
Product Requirements Document (PRD)
Version 1.0  |  April 2026  |  Confidential
Prepared for: Antigravity
Classification: Internal Use Only
1. Executive Summary
The Workforce Daily Tracker is a role-based web application designed to digitise and centralise the daily operations of a 20-person organisation. The system replaces manual spreadsheet-based tracking with a real-time, automated platform that covers employee attendance, task management, leave workflows, performance monitoring, and reporting - all accessible through a browser with no installation required.
This PRD defines every feature, screen, user story, data model, API contract, non-functional requirement, and delivery milestone required to build Version 1.0 of the platform.
Core Objective
Enable Employees, Managers, and HR Admins to perform their daily workforce management tasks from a single, secure, role-aware web application integrated with Microsoft 365.
2. Product Overview
2.1 Product Vision
A unified, real-time workforce management portal that eliminates manual tracking, reduces HR overhead, and gives management live visibility into team productivity and attendance.

4. Functional Requirements
4.1 Authentication & Onboarding
4.1.1 Microsoft SSO Login
Login via Azure Active Directory (AAD) using Microsoft OAuth 2.0 / MSAL
On first login, user profile auto-populated from AAD (name, email, department, manager)
Role assigned by HR Admin post-registration
Session timeout after 8 hours of inactivity
'Remember me' option for 30-day persistent login
4.1.2 User Profile
Editable fields: profile photo, phone number, emergency contact, work location
Read-only fields (sourced from AAD): name, email, department, designation, manager
Password management handled entirely by Microsoft - no local passwords
4.2 Employee Daily Check-In / Check-Out
4.2.1 Check-In Flow
One-click check-in button on employee home screen
System records exact timestamp (IST, server-side)
Status selection: Present / WFH / On-site Client
Optional morning note (max 200 characters)
If check-in missed by 10:00 AM IST  automated reminder notification
Late check-in flag auto-applied if after 09:30 AM IST (configurable threshold)
4.2.2 Check-Out Flow
Check-out button unlocks after minimum 4 hours from check-in
End-of-day summary popup: tasks completed count, self-performance rating (1-5 stars), optional notes
Auto check-out applied at midnight if employee forgets (marked as incomplete)
Total hours worked calculated and stored (check-out time minus check-in time)
4.3 Task Management
4.3.1 Task Assignment (Manager)
Manager opens 'Assign Tasks' panel, selects employee and date
Add tasks with: title (required), description, priority (High / Medium / Low), due time
Tasks can be assigned to multiple employees simultaneously (broadcast)
Edit or delete assigned tasks before employee starts them
View assignment history per employee
4.3.2 Daily Task Log (Employee)
Employee views today's assigned tasks in a checklist format
Mark tasks as: Not Started / In Progress / Completed / Blocked
Add completion notes per task
Add self-initiated tasks not assigned by manager (marked 'Self-assigned')
Tasks carry over to next day if incomplete, with visual overdue indicator
4.3.3 Task Visibility
Manager sees real-time status of all team tasks on a Kanban-style board
HR Admin sees task completion rates across all teams
Filter by employee, date range, status, priority
4.4 Leave Management
4.4.1 Leave Request (Employee)
Select leave type: Full Day / Half Day / Sick Leave / Emergency Leave
Select date range (up to 14 consecutive days per request)
Add reason / supporting notes
View remaining leave balance by type
Cancel pending or approved future leave
4.4.2 Leave Approval Workflow
On submission, Manager receives in-app notification + email
Manager approves, rejects, or requests modification with comment
Employee notified of decision in real-time
On approval: attendance record auto-updated; leave balance decremented
HR Admin can override any manager decision
Leave calendar view shows team availability for conflict detection
4.4.3 Leave Balances
HR Admin configures annual leave entitlements per employee
System tracks: earned, taken, pending, remaining per leave type
Leave year reset on configurable date (default: April 1st)
4.5 Notifications & Reminders
4.5.1 In-App Notifications
Real-time notification bell icon with unread count badge
Notification types: check-in reminder, task assigned, task overdue, leave approved/rejected, manager comment, system alert
Mark as read individually or all at once
Notification history retained for 90 days
4.5.2 Email Notifications
All notifications also sent to user's Microsoft 365 email
Daily digest email at 8:30 AM IST: today's tasks, leave status, reminders
Weekly summary email to managers every Monday: team attendance, completion rates
HR Admin receives weekly workforce summary report via email
4.5.3 Teams Integration (optional, Phase 1.5)
Post daily reminders to a designated MS Teams channel
Leave approval notifications via Teams adaptive cards
4.6 HR / Manager Dashboard
4.6.1 HR Admin Dashboard
KPI cards at top: Total Present Today, WFH, On Leave, Absent, Avg Team Rating
Attendance heatmap: all employees x last 30 days (colour-coded by status)
Performance trend chart: team average rating per week (line chart)
Task completion rate bar chart: per employee, current week
Leave calendar: who is on leave when (month view)
Late arrival tracker: employees with 3+ late check-ins in current month
Top performers widget: ranked by avg rating + task completion %
4.6.2 Manager Dashboard
Filtered view of the above, limited to direct reports only
Today's team status at a glance: attendance + task progress
Pending leave approvals widget with quick approve/reject buttons
Overdue tasks alert: team tasks past due time
4.6.3 Employee Dashboard (My Day)
Today's check-in status and work hours so far
My tasks list for today (from manager + self-assigned)
My leave balance summary
My performance trend (last 30 days self-ratings)
Announcements from HR

5. User Stories
5.1 Employee Stories
As an Employee, I want to check in with one click so that my attendance is recorded without filling a form.
As an Employee, I want to see my assigned tasks for today so that I know what to work on.
As an Employee, I want to mark a task as completed and add a note so that my manager knows my progress.
As an Employee, I want to apply for leave and track its approval status so that I can plan my schedule.
As an Employee, I want to see my monthly attendance and performance trend so that I can improve.
5.2 Manager Stories
As a Manager, I want to assign tasks to my team daily so that everyone has clear deliverables.
As a Manager, I want to see a live view of my team's check-in status at 9 AM so that I know who is present.
As a Manager, I want to approve or reject leave requests with a comment so that the employee is informed.
As a Manager, I want to add a comment on an employee's daily log so that I can give real-time feedback.
As a Manager, I want a weekly report of my team's task completion rates so that I can identify productivity gaps.
5.3 HR Admin Stories
As HR Admin, I want to see a real-time dashboard of all employees' attendance today so that I can act on absences immediately.
As HR Admin, I want to export a monthly attendance report as Excel so that I can process payroll.
As HR Admin, I want to onboard a new employee in under 5 minutes so that they can start using the system on day one.
As HR Admin, I want to configure the leave policy and holidays once per year so that the system runs automatically.
As HR Admin, I want to see who the top and bottom performers are by rating so that I can plan appraisals.

7.3 API Design
REST API Endpoints (summary)
POST /auth/login - Exchange AAD token for app JWT
GET/PUT /users/:id - Get or update user profile
POST /attendance/checkin - Record check-in with status and note
PUT /attendance/checkout/:id - Record check-out with rating and summary
GET /attendance - Query attendance logs (filterable by user, date range, status)
GET/POST /tasks - Get task list or create new task
PUT /tasks/:id - Update task status or details
GET/POST /leave - Get leave requests or submit new request
PUT /leave/:id/approve - Approve or reject leave with comment
GET /dashboard/hr - Aggregated HR dashboard data
GET /dashboard/manager - Manager team summary data
GET /reports/:type - Generate report with filters
POST /reports/:type/export - Download Excel or PDF
GET/POST/PUT/DELETE /employees - Employee management CRUD
GET/PUT /settings - System configuration read/write
7.4 Security
All API endpoints protected by JWT bearer token validation
Role-based access control (RBAC) enforced server-side on every endpoint
HTTPS enforced - HTTP redirected to HTTPS
Azure AD MFA support for all users
Data encrypted at rest (Azure DB encryption) and in transit (TLS 1.3)
Input validation and sanitisation on all user inputs
Rate limiting: 100 requests/minute per user
Audit log: all data-modifying actions logged with user, timestamp, before/after state
GDPR-ready: data export and deletion endpoints available
